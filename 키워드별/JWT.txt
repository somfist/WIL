1. 쿠키 & 세션
	1. 세션은 서버에 가지고 있는 저장소 / 쿠키는 클라이언트가 가지고 있는 임시 저장소
	2. 쿠키가 노출되면 id/pw 같은 민감한 정보가 노출되어 보안에 좋지않음
	3. 쿠키는 사이즈가 제한되어있어 많은 데이터를 담을 수 없음(4kb)
	4. 세션 사용시 세션의 ID(방번호같은느낌)을 쿠키에 담아서 보내주는데 사용자가 많아지면 그만큼 세션을 많이 파야되기에 서버에 부담이 간다
2. JWT
	1. 프리패스권 같은 느낌
	2. 서버에서 서버의 secretKey와 서버에 온 사용자의 식별가능 정보를 담아서 토큰(동전같은느낌?)을 만들어서 보내준다
	3. 다음에 방문할때도 동전을 들고오면 서버의 문을 열어줌
	4. 서버에 따로 세션을 만들어 두지 않아도 되기에 서버에 부담이 덜하다

배운점 => 평상시에 쿠키&세션을 많이 썼었는데 아직 사용자가 많아지는 경우를 많이 못봐서 큰 문제점을 못느꼈었지만 해당 내용에 대해 조금씩 공부하다보니 JWT가 나은거 같다
